# Privacy Policy — LeakShield AI

LeakShield AI is engineered to deliver a premium, transparent, and **privacy-first personal digital footprint review**. Below we document exactly how user data is handled and protected.

---

## 🛡️ Ephemeral Data Processing Guarantees

In this demo/MVP version (`v0.1.0-ai-native-demo`):

1. **Simulated Sandbox:** Every data breach, public footprint listing, inactive account report, and data broker suppress entry represented in the application is **fully fictional and simulated**.
2. **Zero Persistent Tracking:** Any data entered in the setup forms (monitored names, emails, usernames) is kept strictly inside in-memory React states. We do **not** write to `localStorage`, `sessionStorage`, or cookies. Refreshing the browser instantly wipes all session telemetry.
3. **No External Transmissions:** FORM inputs are never transmitted to backend servers or third-party AI APIs. Form structures operate strictly as safe front-end models.
4. **No Password Storage:** LeakShield AI does not capture, store, or transmit passwords in any plaintext or raw form.

---

## ⚖️ GDPR, CCPA & ARCO Compliance blue-printing

LeakShield AI features an **AI Deletion Request Generator** designed to help users exercise their legal rights (under GDPR, CCPA, and Mexican ARCO rights):

* **Human-in-the-Loop:** All opt-out letters, suppress forms, and deletion drafts generated by the AI Copilot are presented to the user for manual review. No automated legal requests are ever submitted to brokers without human verification.
* **Consent-Led Onboarding:** A mandatory consent screen restricts progress until the user explicitly agrees to own or have authorization over monitored details and acknowledges the simulated, non-advisory nature of the AI.

---

## 🚀 Privacy-by-Design Architecture for Production

When transitioning this React application to a production environment with real databases (e.g., Supabase/Firebase) and live APIs (Have I Been Pwned, Vertex AI):

* **Serverless Secrets Proxy:** All external queries must route through authenticated serverless endpoints. No client-side code will carry credentials or execute model invocations directly.
* **Anonymized Logs:** Backend access audit logs will discard PII (Personally Identifiable Information), rate-limiting user requests strictly by anonymized IP hashes.
* **k-Anonymity password checks:** Password compromise checks will exclusively query HIBP using the secure, first-5-character SHA-1 hash prefix protocol.
